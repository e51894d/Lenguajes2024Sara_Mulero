Validación XML Schema (XSD)

XML Schema Definition (XSD) es un lenguaje basado en XML que define la estructura, contenido y restricciones de documentos XML. Su propósito es garantizar que un archivo XML cumpla con un conjunto de reglas predefinidas, lo que facilita la validación de datos y la interoperabilidad entre sistemas.

A diferencia de Document Type Definition (DTD), XSD tiene una sintaxis basada en XML, lo que lo hace más expresivo y compatible con herramientas modernas. Además, permite definir tipos de datos más precisos, anidamientos complejos y restricciones avanzadas sobre los datos.

Estructura Básica de un XSD

Un archivo XSD suele tener la extensión .xsd y se asocia con documentos XML mediante el atributo xsi:noNamespaceSchemaLocation en la cabecera del archivo XML.

Ejemplo de asociación de un XML con su esquema:

                <?xml version="1.0" encoding="UTF-8"?>
                <persona xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
                <xsi:noNamespaceSchemaLocation="persona.xsd">
                <nombre>Benito Pérez G.</nombre>
                <edad>67</edad>
                </persona>

El archivo XSD correspondiente define la estructura válida de este documento.

Elementos Clave en XSD

Elementos Simples

Los elementos simples contienen solo texto y pueden estar restringidos a tipos de datos específicos como cadenas, números o fechas. Ejemplo de definición de elementos simples:

                    <xs:element name="nombre" type="xs:string"/>
                    <xs:element name="edad" type="xs:integer"/>
Elementos Complejos

Los elementos complejos pueden contener otros elementos y atributos. Permiten organizar datos en estructuras jerárquicas.

                    <xs:complexType name="Persona"/>
                        <xs:sequence/>
                            <xs:element name="nombre" type="xs:string"/>
                            <xs:element name="edad" type="xs:integer"/>
                        </xs:sequence/>
                    <xs:complexType/>
Restricciones y Validación

Las restricciones (facetas) permiten definir reglas sobre los valores de los elementos, como rangos numéricos, longitudes de cadena o patrones específicos.

Ejemplo de restricción en la edad (entre 0 y 120):

                    <xs:element name="edad"/>
                    <xs:simpleType>
                    <xs:restriction base="xs:integer"/>
                    <xs:minInclusive value="0"/>
                    <xs:maxInclusive value="120"/>
                    </xs:restriction/>
                    </xs:simpleType/>
                    </xs:element/>

Uso de atributos

XSD permite definir atributos para elementos, lo que agrega información adicional sin afectar la estructura de los datos.

                    <xs:attribute name="id" type="xs:string" use="optional"/>

Herencia y reutilización de definiciones

Los XSD pueden incluir documentación interna para describir el propósito de los elementos y atributos mediante xs:annotation y xs:documentation.

Ejemplo de documentación dentro de un XSD:

                    <xs:annotation>
                    <xs:documentation>Define la estructura de una persona en el sistema.</xs:documentation>
                    </xs:annotation>

Existen herramientas como XMLSpy, Oxygen XML Editor y Notepad++ que permiten crear, editar y validar archivos XSD de manera sencilla. Además, lenguajes como Java y .NET incluyen bibliotecas para validar XML con XSD, como javax.xml.validation en Java.
